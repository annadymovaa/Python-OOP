# Эволюция Python-проекта (ООП)

В этом проекте я реализовал(а) программу для анализа данных, которая поэтапно развивалась от простого чтения файла до полноценного модуля с логированием и внешними интеграциями. Проект демонстрирует глубокое понимание принципов объектно-ориентированного программирования (ООП) в Python, включая инкапсуляцию, наследование, композицию и рефакторинг.

## Структура проекта

*   **src/** — содержит финальные версии исходных файлов проекта.
    *   `analytics.py` — основной модуль с классами для работы с данными.
    *   `config.py` — файл конфигурации.
    *   `make_report.py` — точка входа для запуска программы.
    *   `analytics.log` - результат логирования
    *   `report.txt` - текстовый отчет со статистикой по файлу с данными
*   **datasets/** — содержит различные версии датасетов для тестирования.
    *   `data.csv` — исходный датасет с корректной структурой (заголовок + данные).
    *   `data_no_header.csv` — датасет без строки заголовка.
    *   `data_bad_structure.csv` — датасет с некорректной структурой для проверки обработки ошибок.

## Описание проекта

Проект представляет собой эволюцию одного и того же аналитического модуля. Каждый этап разработки добавлял новую функциональность и улучшал архитектуру кода:

1.  **Основы ООП:** создание базового класса для чтения данных.
2.  **Инкапсуляция и методы:** вынос логики в методы и использование конструктора `__init__` для передачи параметров.
3.  **Наследование и композиция:** создание вложенных классов для разделения логики (расчёты) и наследование для расширения функциональности (прогнозы).
4.  **Архитектура:** разделение кода на модуль (`analytics.py`), конфигурацию (`config.py`) и точку входа (`make_report.py`).
5.  **Продакшн-готовность:** добавление логирования для отладки и метод для отправки уведомлений (например, в Telegram).

## Примеры команд для терминала

Для запуска финальной версии программы используется следующая команда:

```bash
cd src
python3 make_report.py
```

## Особенности реализации

*   **Чистая архитектура:** весь код организован в классы и методы, без логики в глобальной области видимости.
*   **Гибкость:** программа принимает путь к файлу как аргумент, что делает её универсальной.
*   **Обработка ошибок:** реализована проверка структуры входного файла. При обнаружении несоответствий программа корректно завершает работу.
*   **Тестируемость:** модульная структура позволяет легко тестировать отдельные компоненты.
